\subsection{Detailed Description}
The game\textquotesingle{}s \hyperlink{class_game_state_manager}{Game\+State\+Manager}. Handles most of the game loop. 

The \hyperlink{class_game_state_manager}{Game\+State\+Manager} is created once and holds a list (map actually) of all gamestates the game has. All gamestates are loaded on startup, which is not memory efficiënt. The \hyperlink{class_game_state_manager}{Game\+State\+Manager} calls its current gamestate\textquotesingle{}s update, draw and handleinput methods 60 times per second during the game loop. When switching gamestates, the new gamestate is first set with the Set\+Next method and eventually switched to with the Switch\+State method. For this, this class holds two \hyperlink{class_game_state}{Game\+State} pointers that point to gamestates in the list. One to the current state that is now running and one to the next state to which the \hyperlink{class_game_state_manager}{Game\+State\+Manager} will soon switch. As soon as the \hyperlink{class_game_state_manager}{Game\+State\+Manager} is made, gamestates should be added to it using the Add\+Game\+State method and immediately after, the Set\+Next and Switch\+State methods should be called.
